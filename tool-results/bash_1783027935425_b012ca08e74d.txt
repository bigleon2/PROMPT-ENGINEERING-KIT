## Bloc C — Prompt / Instructions

### Méta-prompt maître

Le méta-prompt maître définit le comportement fondamental de PEK v4.0. Il est construit dynamiquement par la fonction `buildSystemPrompt()` en fonction du contexte d'exécution et du mode sélectionné.

```typescript
function buildSystemPrompt(context: ExecutionContext): SystemPrompt {
  const { project, mode, complexity } = context;
  
  // Base commune
  const basePrompt = `Tu es un Architecte en Prompt Engineering spécialisé en conversion de projets IA en kits structurés, réutilisables et validés.`;
  
  // Mode spécifique
  let modeSpecific = '';
  if (mode === 'cot') {
    modeSpecific = `Applique le Chain-of-Thought complet en 7 étapes...`;
  } else if (mode === 'chaining') {
    modeSpecific = `Utilise le Prompt Chaining en 4 étapes pipeline...`;
  } else { // hybrid
    modeSpecific = `Détermine dynamiquement la méthodologie optimale selon la complexité...`;
  }
  
  // Complexité spécifique
  const complexityHandling = complexity === 'simple' ? 
    `Bascule automatiquement en Prompt Chaining` : 
    `Applique le Chain-of-Thought complet`;
  
  return `${basePrompt} ${modeSpecific} ${complexityHandling}`;
}
```

### Instructions de traitement

1. **Sélection du mode d'exécution**
   - Analyser la complexité du projet (simple/moyen/complexe)
   - Si complexité = simple : utiliser Prompt Chaining
   - Si complexité = moyen/complexe : utiliser Chain-of-Thought
   - Si mode = hybride : laisser le système choisir dynamiquement

2. **Application du Chain-of-Thought (7 étapes)**
   - Étape 1 : Compréhension approfondie du projet
   - Étape 2 : Décomposition en composants
   - Étape 3 : Extraction des éléments clés
   - Étape 4 : Structuration du kit
   - Étape 5 : Génération des fichiers
   - Étape 6 : Validation qualité (25pts)
   - Étape 7 : Amélioration et versionnage

3. **Application du Prompt Chaining (4 étapes)**
   - Étape 1 : Analyse initiale du projet
   - Étape 2 : Planification du contenu
   - Étape 3 : Génération séquentielle des blocs
   - Étape 4 : Assemblage et finalisation

### Règles métier

1. **Gestion de l'auto-référence**
   - Limite : 3 niveaux maximum d'auto-analyse
   - Détection : Compteur de récursion et analyse de contexte
   - Action : Interruption avec fallback sur documentation standard

2. **Qualité des outputs**
