    54Рєњ    54Рєњ## A0.1 - Absolute Anchor Grid (Replaces Flow Layout)
    55Рєњ    55Рєњ
    56Рєњ    56Рєњ**Old (DEPRECATED):** `Y = previous_element_Y + height + spacing` - if one element overflows, everything below shifts and gets crushed.
    57Рєњ    57Рєњ
    58Рєњ    58Рєњ**New (MANDATORY):** Page height = 100%. Every element group gets an **absolute percentage Y-anchor**. Elements may only grow **within their own bounding box**. They NEVER push or compress other blocks.
    59Рєњ    59Рєњ
    60Рєњ    60Рєњ### Implementation Rule
    61Рєњ    61Рєњ
    62Рєњ    62Рєњ```
    63Рєњ    63Рєњ# WRONG - flow layout (banned)
    64Рєњ    64Рєњy_cursor = PAGE_H - top_margin
    65Рєњ    65Рєњy_cursor -= title_height
    66Рєњ    66Рєњy_cursor -= gap
    67Рєњ    67Рєњy_cursor -= subtitle_height   # Рєљ if title wraps, subtitle gets crushed
    68Рєњ    68Рєњ
    69Рєњ    69Рєњ# RIGHT - absolute anchor grid
    70Рєњ    70РєњANCHOR_TITLE_Y  = H * 0.30
    71Рєњ    71РєњANCHOR_SUMMARY_Y = H * 0.50
    72Рєњ    72РєњANCHOR_META_Y   = H * 0.70
    73Рєњ    73РєњANCHOR_FOOTER_Y = H * 0.90
    74Рєњ    74Рєњ# Each block renders at its anchor, independent of others
    75Рєњ    75Рєњ```
    76Рєњ    76Рєњ
    77Рєњ    77Рєњ### Bounding Box Containment
    78Рєњ    78Рєњ
    79Рєњ    79РєњEach block has a **maximum bounding box** defined by two consecutive anchors:
    80Рєњ    80Рєњ- Block can grow downward within `[own_anchor_Y, next_anchor_Y - min_gap]`
    81Рєњ    81Рєњ- If content exceeds its bounding box Рєњ trigger overflow protection (see Part 3)
    82Рєњ    82Рєњ- Blocks NEVER consume space belonging to adjacent blocks

   148Рєњ   148Рєњ## A0.3 - Z-Index Layer Management
   149Рєњ   149Рєњ
   150Рєњ   150РєњAll cover elements must be rendered in strict layer order. No exceptions.
   151Рєњ   151Рєњ
   152Рєњ   152Рєњ| Layer | Z-Index | Contents | Rules |
   153Рєњ   153Рєњ|-------|---------|----------|-------|
   154Рєњ   154Рєњ| **Layer 0** (base) | 0 | Background fill (white / light gray) | Always rendered first; full page |
   155Рєњ   155Рєњ| **Layer 1** (background) | 1 | Grids, watermark letters, decorative blocks, large clipped graphics | **MUST enable clip-path** - background elements may extend beyond logical bounds but must be clipped to page physical bounds.    156Рєњ   156Рєњ| **Layer 2** (structure) | 2 | Ultra-thin divider lines, sidebars, corner crop marks | Structural guides that define spatial zones |
   157Рєњ   157Рєњ| **Layer 3** (content) | 3 | All readable text content | Rendered last, always on top |
   158Рєњ   158Рєњ
   159Рєњ   159Рєњ### Clip-Path Enforcement
   160Рєњ   160Рєњ
   161Рєњ   161Рєњ```css
   162Рєњ   162Рєњ/* clip background overflow Рђћ ONLY on Layer 1 */
   163Рєњ   163Рєњ.cover-bg-layer {
   164Рєњ   164Рєњ  position: absolute;
   165Рєњ   165Рєњ  inset: 0;
   166Рєњ   166Рєњ  overflow: hidden;  /* MANDATORY */
   167Рєњ   167Рєњ  z-index: 1;
   168Рєњ   168Рєњ}
   169Рєњ   169Рєњ```
   170Рєњ   170Рєњ
   171Рєњ   171Рєњ> Рџа№ИЈ **Clip scope = Layer 1 ONLY.** `overflow: hidden` must ONLY be on the Layer 1 background container. Layer 2 (lines) and Layer 3 (text) containers must NOT have `overflow: hidden`.
   172Рєњ   172Рєњ
   173Рєњ   173Рєњ###  Anti-Clip Bug (Layer 3 Text Truncation Fix)
   174Рєњ   174Рєњ
   175Рєњ   175Рєњ**Symptom:** Cover text is visible but truncated by an invisible boundary.
   176Рєњ   176Рєњ
   177Рєњ   177Рєњ**Root cause:** `overflow: hidden` scope too broad, clipping text in Layer 3.
   178Рєњ   178Рєњ
   179Рєњ   179Рєњ**Iron rule:**
   180Рєњ   180Рєњ```html
   181Рєњ   181Рєњ<!-- РюЁ CORRECT - overflow:hidden only on Layer 1 -->
   182Рєњ   182Рєњ<div class="cover-layer-1" style="position:absolute; inset:0; overflow:hidden; z-index:1;">
   183Рєњ   183Рєњ  <!-- Background decorative elements -->
   184Рєњ   184Рєњ</div>
   185Рєњ   185Рєњ<div class="cover-layer-2" style="position:absolute; inset:0; z-index:2;">
   186Рєњ   186Рєњ  <!-- Structure lines, no overflow:hidden -->
   187Рєњ   187Рєњ</div>
   188Рєњ   188Рєњ<div class="cover-layer-3" style="position:absolute; inset:0; z-index:3;">
   189Рєњ   189Рєњ  <!-- Text content, no overflow:hidden -->
   190Рєњ   190Рєњ</div>
   191Рєњ   191Рєњ
   192Рєњ   192Рєњ<!-- РЮї WRONG - global overflow:hidden clips text -->

   194Рєњ   194Рєњ  <div class="layer-1">...</div>
   195Рєњ   195Рєњ  <div class="layer-2">...</div>
   196Рєњ   196Рєњ  <div class="layer-3">...</div>  <!-- Text gets clipped! -->
   197Рєњ   197Рєњ</div>

   213Рєњ   213Рєњ###  Minimum Spacing Between Decorative Lines and Text (Line-to-Text Spacing)
   214Рєњ   214Рєњ
   215Рєњ   215Рєњ**Symptom:** Decorative lines on the cover (Layer 2 dividers, corner marks, sidebar edges) are flush against or overlapping with text.
   216Рєњ   216Рєњ
   217Рєњ   217Рєњ**Iron rule:**
   218Рєњ   218Рєњ```

   336Рєњ   336Рєњ
   337Рєњ   337Рєњ**Рџа№ИЈ CJK 3-line titles must reduce to ~42pt.** Do NOT use 50-80pt for 3-line CJK titles Рђћ it WILL overlap the Summary zone.
   338Рєњ   338Рєњ
   339Рєњ   339Рєњ---
   340Рєњ   340Рєњ
   341Рєњ   341Рєњ## S3.2 - Zone Collision Detection
   342Рєњ   342Рєњ
   343Рєњ   343РєњAfter placing each text block, check if its bottom edge penetrates the next zone's top:
   344Рєњ   344Рєњ
   345Рєњ   345Рєњ1. **Reposition first:** Push the collided zone down by `overlap + 20px` gap. All downstream zones shift together.
   346Рєњ   346Рєњ2. **Font reduction (if reposition insufficient):** Decrease title by 2pt increments. Floor: **32pt** for titles, 14pt for meta/summary.
   347Рєњ   347Рєњ3. **Truncation (last resort):** If font reduction hits floor and still collides, truncate with `...`
   348Рєњ   348Рєњ
   349Рєњ   349Рєњ---
   350Рєњ   350Рєњ
   351Рєњ   351Рєњ## S3.3 - Uppercase Lock
   352Рєњ   352Рєњ
   353Рєњ   353Рєњ**Force-uppercase when content is English/Latin:**
   354Рєњ   354Рєњ- Kicker (category label / lead-in text)
   355Рєњ   355Рєњ- Footer (closing date / document number)
   356Рєњ   356Рєњ- Background watermark text
   357Рєњ   357Рєњ- Any Layer 1 decorative text
   358Рєњ   358Рєњ
   359Рєњ   359РєњCJK text is exempt. Mixed CJK+Latin strings: uppercase only the Latin portions. Use `text-transform: uppercase` in CSS or `.upper()` in code.
   360Рєњ   360Рєњ
   361Рєњ   361Рєњ---
   362Рєњ   362Рєњ
   363Рєњ   363Рєњ## S3.4 - Hard Width Boundary Enforcement 
   364Рєњ   364Рєњ
   365Рєњ   365Рєњ**Every drawer/zone has a maximum width. Text MUST respect this width exactly.** Use CSS `max-width` on text containers.
   366Рєњ   366Рєњ
   367Рєњ   367РєњVertical growth (more lines) is acceptable. Horizontal overflow is a **critical bug**.
   368Рєњ   368Рєњ
   369Рєњ   369Рєњ---
   370Рєњ   370Рєњ
   371Рєњ   371Рєњ## S3.5 - Mandatory Summary Auto-Generation 
   372Рєњ   372Рєњ
   373Рєњ   373Рєњ**If the user provides only a title and no description/summary, auto-generate a 2-4 line placeholder.** A title-only cover looks barren. The Summary drawer physically fills mid-page void.
   374Рєњ   374Рєњ
   375Рєњ   375Рєњ---
   376Рєњ   376Рєњ
   377Рєњ   377Рєњ## S3.6 - Background Watermark Full-Display Enforcement 
   378Рєњ   378Рєњ
   379Рєњ   379Рєњ**All watermark text in Layer 1 must be 100% within the visible page area. Cropping is strictly forbidden.**
   380Рєњ   380Рєњ
   381Рєњ   381Рєњ**Applicable scope:**
   382Рєњ   382Рєњ- `cover-backgrounds.md` Recipe 2.1 giant sidebar pillar
   383Рєњ   383Рєњ- `cover-backgrounds.md` Recipe 2.2 bottom full-size text
   384Рєњ   384Рєњ- Any other decorative text in the background layer
   385Рєњ   385Рєњ
   386Рєњ   386Рєњ**Rules:**
   387Рєњ   387Рєњ1. Horizontal text: rendered width must not exceed `W * 0.90` (5% safety margin each side)
   388Рєњ   388Рєњ2. Vertical/rotated text: rendered height must not exceed `H * 0.85` (7.5% safety margin top/bottom)
   389Рєњ   389Рєњ3. If exceeded, scale down font size proportionally Рђћ never truncate
   390Рєњ   390Рєњ4. Anchor coordinates must never exceed page boundaries
   391Рєњ   391Рєњ
   392Рєњ   392Рєњ---
   393Рєњ   393Рєњ
   394Рєњ   394Рєњ## S3.7 - Line-Length Alignment (Line Must Match Text Span)
   395Рєњ   395Рєњ
   396Рєњ   396Рєњ**Decorative lines must be sized relative to the text they serve:**
   397Рєњ   397Рєњ
   398Рєњ   398Рєњ- **Vertical lines:** height = text block height (┬▒ 1U padding above/below first/last element)
   399Рєњ   399Рєњ- **Horizontal lines:** width РЅЦ widest text element in its zone (up to 120%, NEVER shorter)
   400Рєњ   400Рєњ
   401Рєњ   401Рєњ```css
   402Рєњ   402Рєњ.vline {
   403Рєњ   403Рєњ  position: absolute;
   404Рєњ   404Рєњ  top: var(--content-top);     /* align with first text element */
   405Рєњ   405Рєњ  bottom: var(--content-bottom); /* align with last text element */
   406Рєњ   406Рєњ}
   407Рєњ   407Рєњ.hline {
   408Рєњ   408Рєњ  width: max(100%, 200px);     /* at least as wide as parent text container */
   409Рєњ   409Рєњ}
   410Рєњ   410Рєњ```
   411Рєњ   411Рєњ
   412Рєњ   412Рєњ**Checklist:**
   413Рєњ   413Рєњ- [ ] Every vertical lineРђЎs height matches its adjacent text block span
   414Рєњ   414Рєњ- [ ] Every horizontal lineРђЎs width РЅЦ widest text element in its zone
   415Рєњ   415Рєњ- [ ] No decorative line is shorter than the text it accompanies
   416Рєњ   416Рєњ
   417Рєњ   417Рєњ---
   418Рєњ   418Рєњ
   419Рєњ   419Рєњ## S3.8 - Vertical Balance (Anti-Top-Heavy Layout)
   420Рєњ   420Рєњ
   421Рєњ   421Рєњ**Problem:** Content clusters at top of page, bottom 40%+ is dead whitespace.
   422Рєњ   422Рєњ
   423Рєњ   423Рєњ**Rule:** When total content height < 50% of page height, switch to centered distribution (flexbox `justify-content: center`). Otherwise use anchor grid:
   424Рєњ   424Рєњ
   425Рєњ   425Рєњ| Content Volume | Title Anchor | Summary Anchor | Meta Anchor |
   426Рєњ   426Рєњ|---------------|-------------|----------------|-------------|
   427Рєњ   427Рєњ| **Sparse** (fill < 50%) | Centered mode | Centered mode | Centered mode |
   428Рєњ   428Рєњ| **Normal** (fill 50-80%) | `H * 0.30` | `H * 0.48` | `H * 0.70` |
   429Рєњ   429Рєњ| **Dense** (fill > 80%) | `H * 0.20` | `H * 0.40` | `H * 0.65` |
   430Рєњ   430Рєњ
   431Рєњ   431Рєњ**CJK Title Size Compensation:** CJK characters appear smaller at same pt size. Increase by 15-20%:
   432Рєњ   432Рєњ- CJK Hero Title: 50-80pt (Latin: 45-65pt)
   433Рєњ   433Рєњ- CJK Kicker: 11-12pt (Latin: 9pt)
   434Рєњ   434Рєњ- CJK Summary: 17-20pt (Latin: 16-18pt)
   435Рєњ   435Рєњ
   436Рєњ   436Рєњ**Checklist:**
   437Рєњ   437Рєњ- [ ] No cover has >40% dead whitespace at the bottom
   438Рєњ   438Рєњ- [ ] CJK titles are 15-20% larger than equivalent Latin titles
   439Рєњ   439Рєњ- [ ] Sparse-content covers use centered distribution
   440Рєњ   440Рєњ
   441Рєњ   441Рєњ---
   442Рєњ   442Рєњ
   443Рєњ   443Рєњ## S3.9 - Percentage Positioning Requires Known-Size Container
   444Рєњ   444Рєњ
   445Рєњ   445Рєњ**Problem:** `top: XX%` resolves against containing blockРђЎs height. If height is undefined (all children absolutely positioned), percentages collapse and elements stack.
   446Рєњ   446Рєњ
   447Рєњ   447Рєњ**Iron rule:** Container with `top: XX%` children MUST have deterministic height (`height: 100%`, `inset: 0`, or `top` + `bottom` pair).
   448Рєњ   448Рєњ
   449Рєњ   449Рєњ```css
   450Рєњ   450Рєњ/* РюЁ Safest: flat structure with px values */
   451Рєњ   451Рєњ.cover { position: relative; width: 794px; height: 1123px; }
   452Рєњ   452Рєњ.kicker { position: absolute; top: 225px; left: 95px; }
   453Рєњ   453Рєњ.title  { position: absolute; top: 292px; left: 95px; }
   454Рєњ   454Рєњ
   455Рєњ   455Рєњ/* РюЁ OK: wrapper with inset:0 gives it known height */
   456Рєњ   456Рєњ.content-left { position: absolute; inset: 0; width: 55%; }
   457Рєњ   457Рєњ.title { position: absolute; top: 26%; }  /* 26% of 1123px РюЊ */
   458Рєњ   458Рєњ
   459Рєњ   459Рєњ/* РЮї BANNED: wrapper without height, percentage is undefined */
   460Рєњ   460Рєњ.content-left { position: absolute; left: 12%; top: 0; width: 55%; }
   461Рєњ   461Рєњ.title { position: absolute; top: 26%; }  /* 26% of WHAT? Рєњ overlap */
   462Рєњ   462Рєњ```
   463Рєњ   463Рєњ
