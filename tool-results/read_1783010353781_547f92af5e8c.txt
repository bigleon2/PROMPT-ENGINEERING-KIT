   216вҶ’   216вҶ’   216вҶ’## Step 3: EDIT - Content Transformation
   217вҶ’   217вҶ’   217вҶ’
   218вҶ’   218вҶ’   218вҶ’**Before writing any code, restructure raw text into visual roles.** This is the most impactful quality step - skipping it produces "Word document with a PDF extension".
   219вҶ’   219вҶ’   219вҶ’
   220вҶ’   220вҶ’   220вҶ’### 3a. Typographic Role Extraction
   221вҶ’   221вҶ’   221вҶ’
   222вҶ’   222вҶ’   222вҶ’Parse raw text and categorize content into visual roles:
   223вҶ’   223вҶ’   223вҶ’
   224вҶ’   224вҶ’   224вҶ’| Role | ReportLab Mapping | Description |
   225вҶ’   225вҶ’   225вҶ’|------|------------------|-------------|
   226вҶ’   226вҶ’   226вҶ’| **Hero / Display** | Cover title (36-42pt) or section opener | 1-5 words, the emotional hook |
   227вҶ’   227вҶ’   227вҶ’| **Kicker / Eyebrow** | Subtitle or section intro (10-12pt, muted) | Tiny context line above main heading |
   228вҶ’   228вҶ’   228вҶ’| **Data Sculpture** | CalloutBox or bold stat block | Extract impactful numbers (97%, $4.2M, -45ms) |
   229вҶ’   229вҶ’   229вҶ’| **Pull Quote** | BlockQuote (italic + left indent 24pt + accent border) | Most provocative sentence, standalone |
   230вҶ’   230вҶ’   230вҶ’| **Body** | Standard paragraph | Everything else |
   231вҶ’   231вҶ’   231вҶ’
   232вҶ’   232вҶ’   232вҶ’**Rule**: Before writing any body paragraph, scan for numbers with units. Every metric like "revenue grew by 12%" or "latency dropped to 45ms" MUST be extracted into a CalloutBox or metrics table - never buried in paragraph prose. See В§Data-to-Ink Ratio Rules below.
   233вҶ’   233вҶ’   233вҶ’
   234вҶ’   234вҶ’   234вҶ’### 3b. Section Pacing
   235вҶ’   235вҶ’   235вҶ’
   236вҶ’   236вҶ’   236вҶ’- If any H1 section has **>400 words** of continuous body text without visual breaks, split into H2 subsections or insert a visual break (table, chart, callout)
   237вҶ’   237вҶ’   237вҶ’- Aim for at least 1 visual element (table/chart/callout) per 2-3 pages of body text
   238вҶ’   238вҶ’   238вҶ’- Dense text walls are the #1 sign of poor report design
   239вҶ’   239вҶ’   239вҶ’
   240вҶ’   240вҶ’   240вҶ’### 3c. Content Sanitization for External Sources (MANDATORY when source is PDF/OCR/web)
   241вҶ’   241вҶ’   241вҶ’
   242вҶ’   242вҶ’   242вҶ’When content is extracted from an external PDF, OCR output, or web scrape, it may contain invisible garbage characters that cause garbled rendering. **You MUST sanitize the text content before writing it into any `Paragraph()` or `canvas.drawString()`.**
   243вҶ’   243вҶ’   243вҶ’
   244вҶ’   244вҶ’   244вҶ’This is different from `code.sanitize` (which cleans Python source code). Content sanitization cleans the **text data itself**.
   245вҶ’   245вҶ’   245вҶ’
   246вҶ’   246вҶ’   246вҶ’```python
   247вҶ’   247вҶ’   247вҶ’import re
   248вҶ’   248вҶ’   248вҶ’
   249вҶ’   249вҶ’   249вҶ’def content_sanitize(text: str) -> str:
   250вҶ’   250вҶ’   250вҶ’    """Strip invisible/control chars from externally-sourced text."""
   251вҶ’   251вҶ’   251вҶ’    # Remove control chars (except \n \t)
   252вҶ’   252вҶ’   252вҶ’    text = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f\x7f]', '', text)
   253вҶ’   253вҶ’   253вҶ’    # Remove zero-width chars
   254вҶ’   254вҶ’   254вҶ’    text = re.sub(r'[\u200b-\u200f\u2028-\u202f\u2060\ufeff]', '', text)
   255вҶ’   255вҶ’   255вҶ’    # Remove replacement char (upstream encoding error)
   256вҶ’   256вҶ’   256вҶ’    text = text.replace('\ufffd', '')
   257вҶ’   257вҶ’   257вҶ’    # Remove variation selectors
   258вҶ’   258вҶ’   258вҶ’    text = re.sub(r'[\ufe00-\ufe0f]', '', text)
   259вҶ’   259вҶ’   259вҶ’    # Remove private use area chars
   260вҶ’   260вҶ’   260вҶ’    text = re.sub(r'[\ue000-\uf8ff]', '', text)
   261вҶ’   261вҶ’   261вҶ’    return text
   262вҶ’   262вҶ’   262вҶ’```
   263вҶ’   263вҶ’   263вҶ’
   264вҶ’   264вҶ’   264вҶ’**When to use:**
   265вҶ’   265вҶ’   265вҶ’| Source | Must sanitize? |
   266вҶ’   266вҶ’   266вҶ’|--------|---------------|
   267вҶ’   267вҶ’   267вҶ’| User typed text in prompt | No (usually clean) |
   268вҶ’   268вҶ’   268вҶ’| Extracted from PDF via `pdf.py extract.text` | **YES** |
   269вҶ’   269вҶ’   269вҶ’| OCR output | **YES** |
   270вҶ’   270вҶ’   270вҶ’| Web scrape / HTML strip | **YES** |

   302вҶ’   302вҶ’   302вҶ’## Character Safety Rule (MANDATORY)
   303вҶ’   303вҶ’   303вҶ’
   304вҶ’   304вҶ’   304вҶ’Three rules for safe character handling in ReportLab PDFs:
   305вҶ’   305вҶ’   305вҶ’
   306вҶ’   306вҶ’   306вҶ’**a) Superscripts and subscripts**: Use `<super>`/`<sub>` tags, never raw Unicode superscript/subscript characters (e.g., `\u00b2`, `\u2082`).
   307вҶ’   307вҶ’   307вҶ’
   308вҶ’   308вҶ’   308вҶ’**b) Emoji**: ReportLab cannot render emoji. If content contains emoji, use Creative brief (HTML + Playwright).
   309вҶ’   309вҶ’   309вҶ’
   310вҶ’   310вҶ’   310вҶ’**c) Font fallback for mixed CJK/Latin text (MANDATORY)**: After registering fonts, you **MUST** call `install_font_fallback()` once. This automatically wraps missing-glyph characters in `<font>` tags inside every `Paragraph()`. No manual `<font name="...">` wrapping needed for mixed Chinese-English text. **Skipping this call is the #1 cause of Helvetica garbled text in CJK documents.**
   311вҶ’   311вҶ’   311вҶ’
   312вҶ’   312вҶ’   312вҶ’**d) canvas.drawString / drawRightString CJK font rule (MANDATORY)**: `install_font_fallback()` only works on `Paragraph()` objects. It does NOT affect `canvas.drawString()` / `canvas.drawRightString()` / `canvas.drawCentredString()`. If the text passed to any `canvas.draw*String()` call contains CJK characters (Chinese/Japanese/Korean), you **MUST** call `canvas.setFont('NotoSerifSC', size)` (or another registered CJK font) before drawing. Using FreeSerif / Helvetica / Times for CJK canvas text = garbled output.
   313вҶ’   313вҶ’   313вҶ’
   314вҶ’   314вҶ’   314вҶ’```python
   315вҶ’   315вҶ’   315вҶ’# вқҢ WRONG вҖ” FreeSerif has no CJK glyphs, 'з§‘еӨ§и®ҜйЈһ' will be garbled
   316вҶ’   316вҶ’   316вҶ’canvas.setFont('FreeSerif-Italic', 8)
   317вҶ’   317вҶ’   317вҶ’canvas.drawRightString(x, y, 'SZSE: 002230  В·  з§‘еӨ§и®ҜйЈһ')
   318вҶ’   318вҶ’   318вҶ’
   319вҶ’   319вҶ’   319вҶ’# вң… CORRECT вҖ” use CJK font for mixed text
   320вҶ’   320вҶ’   320вҶ’canvas.setFont('NotoSerifSC', 8)
   321вҶ’   321вҶ’   321вҶ’canvas.drawRightString(x, y, 'SZSE: 002230  В·  з§‘еӨ§и®ҜйЈһ')
   322вҶ’   322вҶ’   322вҶ’
   323вҶ’   323вҶ’   323вҶ’# вң… ALSO CORRECT вҖ” pure English text can use FreeSerif
   324вҶ’   324вҶ’   324вҶ’canvas.setFont('FreeSerif-Italic', 8)
   325вҶ’   325вҶ’   325вҶ’canvas.drawString(x, y, 'Prepared by Strategy Advisory Team')
   326вҶ’   326вҶ’   326вҶ’```

   380вҶ’   380вҶ’   380вҶ’> "Does this string contain a character outside basic CJK or Mathematical/relational operators?"
   381вҶ’   381вҶ’   381вҶ’> If YES вҶ’ it MUST be inside a `Paragraph()` with the appropriate tag.

   387вҶ’   387вҶ’   387вҶ’> "Does this string contain Japanese kana (гҒ®, гҒҢ, гҒҜ etc.) or rare Unicode symbols?"
