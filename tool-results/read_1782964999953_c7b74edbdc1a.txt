   229→   278→   278→### 4.2 Protocole — 13 Étapes Séquentielles
   230→   279→   279→
   231→   280→   280→#### Étape 1 — Collecte et Analyse des Demandes
   232→   281→   281→**Objectif** : Relire toute la conversation pour extraire chaque demande explicite et implicite.
   233→   282→   282→- Relire tous les messages utilisateur (demandes initiales, specs, contraintes, corrections)
   234→   283→   283→- Identifier demandes explicites (ce que l'utilisateur a directement demandé)
   235→   284→   284→- Identifier demandes implicites (prérequis, effets de bord, conséquences logiques)
   236→   285→   285→- Lister chaque demande sans ambiguïté
   237→   286→   286→- **Méthode** : Lire conversation complète. Utiliser worklog.md si disponible.
   238→   287→   287→
   239→   288→   288→#### Étape 2 — Lecture et Analyse du Projet
   240→   289→   289→**Objectif** : Traverser tous les fichiers du projet pour comprendre structure, architecture, dépendances, état actuel.
   241→   290→   290→- **Principe** : Un plan sans connaissance du projet est générique et probablement inadéquat.
   242→   291→   291→- Structure fichiers : arborescence, configs (package.json, tsconfig, etc.)
   243→   292→   292→- Code source : chaque fichier significatif
   244→   293→   293→- Schéma base de données (si applicable)
   245→   294→   294→- Dépendances : imports inter-modules, dépendances externes
   246→   295→   295→- Configuration : fichiers config pertinents
   247→   296→   296→- Documentation : README, worklog, specs, fichiers référence
   248→   297→   297→- **Méthode** : Utiliser Read, Glob, Grep pour traverser chaque fichier significatif.
   249→   298→   298→
   250→   299→   299→#### Étape 3 — Identification de la Nature du Projet
   251→   300→   300→**Objectif** : Déterminer type projet, technologies, architecture, complexité.
   252→   301→   301→- Type projet : web app, API, script, document, analyse, skill, etc.
   253→   302→   302→- Technologies : framework, langage, base de données, build tools
   254→   303→   303→- Architecture : monolithe, micro-services, fullstack, frontend only, backend only
   255→   304→   304→- Complexité : nombre fichiers, profondeur arborescence, dépendances
   256→   305→   305→
   257→   306→   306→#### Étape 4 — Identification des Objectifs
   258→   307→   307→**Objectif** : Lister chaque objectif explicitement, sans ambiguïté ni omission.
   259→   308→   308→- Lister objectifs dérivés des demandes utilisateur
   260→   309→   309→- Lister objectifs implicites (prérequis, effets de bord, contraintes)
   261→   310→   310→- Vérifier que chaque objectif est mesurable (atteint ou non)
   262→   311→   311→- Supprimer doublons
   263→   312→   312→
   264→   313→   313→#### Étape 5 — Décomposition en Sous-tâches Adaptées
   265→   314→   314→**Objectif** : Décomposer chaque objectif en sous-tâches atomiques avec ordre logique d'exécution.
   266→   315→   315→- Pour chaque objectif, identifier sous-tâches atomiques
   267→   316→   316→- Ordonner logiquement (prérequis d'abord, puis exécution, puis validation)
   268→   317→   317→- Adapter au type projet :
   269→   318→   318→  - **Fullstack** : inclure vérification API frontend-backend, schéma DB, auth
   270→   319→   319→  - **Frontend only** : inclure vérification responsive, accessibilité, composants
   271→   320→   320→  - **Backend/API** : inclure vérification endpoints, validation, sécurité
   272→   321→   321→  - **Document/PDF** : inclure vérification contenu, layout, cohérence données
   273→   322→   322→  - **Script/automation** : inclure cas limites, robustesse, gestion erreurs
   274→   323→   323→
   275→   324→   324→#### Étape 6 — Détection des Dépendances
   276→   325→   325→**Objectif** : Identifier dépendances entre sous-tâches et contraintes de précédence.
   277→   326→   326→- Dépendances séquentielles (A doit finir avant B)
   278→   327→   327→- Dépendances parallèles (A et B peuvent tourner simultanément)
   279→   328→   328→- Dépendances conditionnelles (B tourne seulement si A réussit/échoue)
   280→   329→   329→- Vérifier absence de dépendances circulaires
   281→   330→   330→- **Méthode** : Construire graphe de dépendances. Identifier chemins critiques.
   282→   331→   331→
   283→   332→   332→#### Étape 7 — Priorisation
   284→   333→   333→**Objectif** : Classer sous-tâches par impact sur résultat final.
   285→   334→   334→
   286→   335→   335→| Priorité | Définition |
   287→   336→   336→|----------|------------||
   288→   337→   337→| **Critique** | Échec invalide le résultat entier |
   289→   338→   338→| **Importante** | Impact significatif sur qualité |
   290→   339→   339→| **Secondaire** | Améliore résultat mais non bloquant |
   291→   340→   340→
   292→   341→   341→#### Étape 8 — Estimation des Risques
   293→   342→   342→**Objectif** : Pour chaque sous-tâche, identifier risques potentiels et plans de fallback.
   294→   343→   343→- Complexité technique : connaissance spécialisée requise ?
   295→   344→   344→- Ambiguïté : clairement définie ou ouverte à interprétation ?
   296→   345→   345→- Dépendance externe : repose sur API, service, outil qui pourrait échouer ?
   297→   346→   346→- Fallback : que faire si sous-tâche échoue ? Contourner, reporter, simplifier ?
   298→   347→   347→
   299→   348→   348→#### Étape 9 — Structuration du Plan
   300→   349→   349→**Objectif** : Produire un plan d'actions formel, complet, adapté au projet.
   301→   350→   350→
   302→   351→   351→Le plan doit inclure :
   303→   352→   352→- **En-tête** : nature projet, objectifs, contraintes
   304→   353→   353→- **Liste ordonnée d'étapes** avec IDs (1, 2-a, 2-b, 3...)
   305→   354→   354→  - Pour chaque étape : objectif, fichiers concernés, dépendances, priorité
   306→   355→   355→  - Indication parallélisme où applicable
   307→   356→   356→  - Critères de validation par étape
   308→   357→   357→- **Carte dépendances** : résumé relations étapes
   309→   358→   358→- **Matrice risques** : risques et plans fallback
   310→   359→   359→
   311→   360→   360→#### Étape 10 — Validation du Plan
   312→   361→   361→**Objectif** : Checklist :
