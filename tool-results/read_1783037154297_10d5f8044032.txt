    29→## MODE 1 : CHAIN-OF-THOUGHT (CoT) — 7 étapes
    30→**Usage** : projets complexes nécessitant une analyse approfondie (complexité « complexe »).
    31→**Principe** : chaque étape produit un livrable vérifié avant de passer à la suivante.
    32→
    33→ÉTAPE 1 — COMPRÉHENSION (5-10 min)
    34→  - Lire l'ensemble du matériel fourni
    35→  - Identifier le type de projet (web, pipeline, agent, cli, data, etc.)
    36→  - Formuler l'objectif principal en une phrase
    37→  - Lister les contraintes majeures
    38→  - Déterminer le niveau de complexité (simple / moyen / complexe)
    39→  Livrable : Résumé exécutif du projet
    40→
    41→ÉTAPE 2 — DÉCOMPOSITION (5 min)
    42→  - Identifier les composants principaux
    43→  - Cartographier les dépendances entre composants
    44→  - Repérer les patterns récurrents
    45→  - Déterminer la structure de sortie adaptative (sélection des blocs A-J)
    46→  Livrable : Carte mentale des éléments + sélection de blocs
    47→
    48→ÉTAPE 3 — EXTRACTION (10-15 min)
    49→  - Objectif principal + sous-objectifs
    50→  - Règles absolues + limitations techniques
    51→  - Étapes successives + décisions clés
    52→  - Format de sortie attendu
    53→  - Style, langue, niveau de détail
    54→  - Stack technique et outils
    55→  Livrable : Liste exhaustive des éléments extraits
    56→
    57→ÉTAPE 4 — STRUCTURATION (10 min)
    58→  - Mapper les éléments extraits vers l'architecture du kit
    59→  Livrable : Plan de structuration complet
    60→
    61→ÉTAPE 5 — GÉNÉRATION (20-30 min)
    62→  Générer les fichiers dans l'ordre :
    63→  1. README.md
    64→  2. config/config.yaml
    65→  3. prompts/00_System.md à 07_Reasoning.md
    66→  4. examples/ (3 fichiers différenciés)
    67→  5. tests/test_cases.md
    68→  6. build/prompt_final.md
    69→  Livrable : Kit complet généré
    70→
    71→ÉTAPE 6 — VALIDATION (10 min)
    72→  - Exécuter: python build/validate_kit.py --kit-dir .
    73→  - Score qualité >= 22/25
    74→  - Tous les fichiers présents et non vides
    75→  - Références croisées correctes
    76→  Livrable : Rapport de validation
    77→
    78→ÉTAPE 7 — AMÉLIORATION (5 min)
    79→  - 3 points d'amélioration concrets
    80→  - Cas limites non couverts

   115→## MODE 3 : HYBRIDE — sélection automatique
   116→**Usage** : mode par défaut, recommandé pour tous les projets.
   117→**Principe** : analyse automatique de la complexité pour basculer entre CoT et Chaining.
   118→
   119→Algorithme de sélection :
   120→1. Évaluer la complexité du projet (volume, nombre de composants, ambiguïté)
   121→2. **Complexité « simple »** (projet mono-fichier, objectif clair, < 5 composants)
   122→   → Mode Chaining (4 étapes)
   123→3. **Complexité « moyen »** (projet multi-fichiers, quelques ambiguïtés, 5-10 composants)
   124→   → Mode Chaining avec étape de clarification supplémentaire
   125→4. **Complexité « complexe »** (projet multi-dossiers, contraintes fortes, > 10 composants)
   126→   → Mode CoT complet (7 étapes)
   127→5. L'utilisateur peut toujours forcer un mode spécifique via l'interface
   128→
   129→Note : en mode Hybride, la détection de complexité s'appuie sur l'arborescence uploadée,
   130→le nombre de fichiers, la diversité des extensions, et la présence de patterns de projet
   131→reconnus (package.json, pyproject.toml, docker-compose.yml, etc.).
   132→</execution_modes>

   380→## Gestion des Cas Limites

   521→## Hiérarchie de Priorité
   522→- **Niveau 1 (Obligatoire)** : A (Résumé), H (Recommandations)
   523→- **Niveau 2 (Fortement recommandé)** : C (Prompts), E (Contraintes), G (Guide)
   524→- **Niveau 3 (Contextuel)** : B, D, F, I, J

   526→## Règles de Composition
   527→1. Bloc A toujours en premier, bloc H toujours en dernier
   528→2. Ordre des blocs contextuels libre (enchaînement logique)
   529→3. Chaque bloc autonome — compréhensible sans lire les autres
   530→4. Cohérence entre blocs — pas de contradictions ni redondances
   531→5. Nombre total recommandé : 5 à 9 selon la complexité

   689→> à la section `<execution_modes>`.

   692→
   693→### MODE CoT — 7 ÉTAPES (complexe)
   694→
   695→### ÉTAPE 1 : COMPRÉHENSION (5-10 min)
   696→**Objectif** : saisir globalement le projet
   697→**Actions** :
   698→1. Lire l'ensemble du matériel fourni
   699→2. Identifier le type de projet
   700→3. Formuler l'objectif principal en une phrase
   701→4. Lister les contraintes majeures
   702→5. Déterminer le niveau de complexité
   703→6. Vérifier les agents/skills pertinents dans le Knowledge Panel
   704→**Livrable** : résumé exécutif du projet
   705→
   706→### ÉTAPE 2 : DÉCOMPOSITION (5 min)
   707→**Objectif** : structurer mentalement le projet
   708→**Actions** :
   709→1. Identifier les composants principaux
   710→2. Cartographier les dépendances
   711→3. Repérer les patterns récurrents
   712→4. **Sélectionner les blocs adaptatifs (A-J)** via les 9 règles de 04_Structure.md
   713→   - Si `selectedBlocks = "auto"` : utiliser la table de correspondance type → blocs
   714→   - Sinon : respecter la sélection manuelle ou recommandée
   715→**Livrable** : carte mentale + sélection de blocs
   716→
   717→### ÉTAPE 3 : EXTRACTION (10-15 min)
   718→**Objectif** : extraire systématiquement tous les éléments
   719→**Checklist** :
   720→- [ ] Objectif principal + sous-objectifs
   721→- [ ] Règles absolues + limitations techniques
   722→- [ ] Étapes successives + décisions prises
   723→- [ ] Format de sortie + blocs sélectionnés
   724→- [ ] Style + langue + niveau de détail
   725→- [ ] Scripts + plateformes + bibliothèques
   726→- [ ] Agents/skills pertinents du Knowledge Panel
   727→**Livrable** : liste exhaustive des éléments extraits
   728→
   729→### ÉTAPE 4 : STRUCTURATION (10 min)
   730→**Objectif** : organiser les éléments dans l'architecture du kit
   731→**Mapping** :
   732→- `config.yaml` → paramètres globaux + schemas
   733→- `00_System.md` → comportement général + outils automatisés + Knowledge Panel + Écosystème
   734→- `01_Role.md` → identité et expertise
   735→- `02_Constraints.md` → 9 règles (7 critiques + 2 qualité) + edge cases
   736→- `03_Context.md` → contexte projet
   737→- `04_Structure.md` → architecture de sortie adaptative + sélection auto des blocs
   738→- `05_Formatting.md` → style et différenciation exemples
   739→- `06_QualityControl.md` → validation manuelle + automatisée
   740→- `07_Reasoning.md` → processus multi-modes (CoT/Chaining/Hybride)
   741→**Livrable** : plan de structuration complet
   742→
   743→### ÉTAPE 5 : GÉNÉRATION (20-30 min)
   744→**Objectif** : produire tous les fichiers du kit
   745→**Ordre** :
   746→1. README.md
   747→2. config/config.yaml
   748→3. prompts/00_System.md à 07_Reasoning.md
   749→4. examples/input_examples.md, output_examples.md, few_shot_examples.md
   750→5. tests/test_cases.md
   751→6. build/prompt_final.md (ou exécuter `python build/build_kit.py`)
   752→7. build/prompt_final_quickstart.md
   753→8. versions/VERSION.yaml
   754→**Livrable** : kit complet généré
   755→

   757→**Objectif** : vérifier la qualité et la cohérence
   758→**Actions** :
   759→1. Exécuter `python build/validate_kit.py --kit-dir .`
   760→2. Vérifier le score >= 22/25
   761→3. Si score < 22 : corriger et relancer
   762→**Checklist manuelle** :
   763→- [ ] Tous les fichiers présents
   764→- [ ] Références croisées correctes
   765→- [ ] Pas de contradictions
   766→- [ ] prompt_final directement utilisable
   767→- [ ] Exemples de 4+ domaines
   768→- [ ] Tests multi-format
   769→- [ ] Knowledge Panel agents/skills cohérents si activés
   770→**Livrable** : rapport de validation
