   191вЖТ   220вЖТ   220вЖТ   220вЖТ### 3a. Typographic Role Extraction
   192вЖТ   221вЖТ   221вЖТ   221вЖТ
   193вЖТ   222вЖТ   222вЖТ   222вЖТParse raw text and categorize content into visual roles:
   194вЖТ   223вЖТ   223вЖТ   223вЖТ
   195вЖТ   224вЖТ   224вЖТ   224вЖТ| Role | ReportLab Mapping | Description |
   196вЖТ   225вЖТ   225вЖТ   225вЖТ|------|------------------|-------------|

   211вЖТ   240вЖТ   240вЖТ   240вЖТ### 3c. Content Sanitization for External Sources (MANDATORY when source is PDF/OCR/web)
   212вЖТ   241вЖТ   241вЖТ   241вЖТ
   213вЖТ   242вЖТ   242вЖТ   242вЖТWhen content is extracted from an external PDF, OCR output, or web scrape, it may contain invisible garbage characters that cause garbled rendering. **You MUST sanitize the text content before writing it into any `Paragraph()` or `canvas.drawString()`.**
   214вЖТ   243вЖТ   243вЖТ   243вЖТ
   215вЖТ   244вЖТ   244вЖТ   244вЖТThis is different from `code.sanitize` (which cleans Python source code). Content sanitization cleans the **text data itself**.
   216вЖТ   245вЖТ   245вЖТ   245вЖТ
   217вЖТ   246вЖТ   246вЖТ   246вЖТ```python
   218вЖТ   247вЖТ   247вЖТ   247вЖТimport re
   219вЖТ   248вЖТ   248вЖТ   248вЖТ
   220вЖТ   249вЖТ   249вЖТ   249вЖТdef content_sanitize(text: str) -> str:
   221вЖТ   250вЖТ   250вЖТ   250вЖТ    """Strip invisible/control chars from externally-sourced text."""
   222вЖТ   251вЖТ   251вЖТ   251вЖТ    # Remove control chars (except \n \t)
   223вЖТ   252вЖТ   252вЖТ   252вЖТ    text = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f\x7f]', '', text)
   224вЖТ   253вЖТ   253вЖТ   253вЖТ    # Remove zero-width chars
   225вЖТ   254вЖТ   254вЖТ   254вЖТ    text = re.sub(r'[\u200b-\u200f\u2028-\u202f\u2060\ufeff]', '', text)
   226вЖТ   255вЖТ   255вЖТ   255вЖТ    # Remove replacement char (upstream encoding error)
   227вЖТ   256вЖТ   256вЖТ   256вЖТ    text = text.replace('\ufffd', '')
   228вЖТ   257вЖТ   257вЖТ   257вЖТ    # Remove variation selectors
   229вЖТ   258вЖТ   258вЖТ   258вЖТ    text = re.sub(r'[\ufe00-\ufe0f]', '', text)
   230вЖТ   259вЖТ   259вЖТ   259вЖТ    # Remove private use area chars
   231вЖТ   260вЖТ   260вЖТ   260вЖТ    text = re.sub(r'[\ue000-\uf8ff]', '', text)
   232вЖТ   261вЖТ   261вЖТ   261вЖТ    return text
   233вЖТ   262вЖТ   262вЖТ   262вЖТ```
   234вЖТ   263вЖТ   263вЖТ   263вЖТ
   235вЖТ   264вЖТ   264вЖТ   264вЖТ**When to use:**
   236вЖТ   265вЖТ   265вЖТ   265вЖТ| Source | Must sanitize? |
   237вЖТ   266вЖТ   266вЖТ   266вЖТ|--------|---------------|
   238вЖТ   267вЖТ   267вЖТ   267вЖТ| User typed text in prompt | No (usually clean) |

   273вЖТ   302вЖТ   302вЖТ   302вЖТ## Character Safety Rule (MANDATORY)
   274вЖТ   303вЖТ   303вЖТ   303вЖТ
   275вЖТ   304вЖТ   304вЖТ   304вЖТThree rules for safe character handling in ReportLab PDFs:
   276вЖТ   305вЖТ   305вЖТ   305вЖТ
   277вЖТ   306вЖТ   306вЖТ   306вЖТ**a) Superscripts and subscripts**: Use `<super>`/`<sub>` tags, never raw Unicode superscript/subscript characters (e.g., `\u00b2`, `\u2082`).
   278вЖТ   307вЖТ   307вЖТ   307вЖТ
   279вЖТ   308вЖТ   308вЖТ   308вЖТ**b) Emoji**: ReportLab cannot render emoji. If content contains emoji, use Creative brief (HTML + Playwright).
   280вЖТ   309вЖТ   309вЖТ   309вЖТ
   281вЖТ   310вЖТ   310вЖТ   310вЖТ**c) Font fallback for mixed CJK/Latin text (MANDATORY)**: After registering fonts, you **MUST** call `install_font_fallback()` once. This automatically wraps missing-glyph characters in `<font>` tags inside every `Paragraph()`. No manual `<font name="...">` wrapping needed for mixed Chinese-English text. **Skipping this call is the #1 cause of Helvetica garbled text in CJK documents.**
   282вЖТ   311вЖТ   311вЖТ   311вЖТ
   283вЖТ   312вЖТ   312вЖТ   312вЖТ**d) canvas.drawString / drawRightString CJK font rule (MANDATORY)**: `install_font_fallback()` only works on `Paragraph()` objects. It does NOT affect `canvas.drawString()` / `canvas.drawRightString()` / `canvas.drawCentredString()`. If the text passed to any `canvas.draw*String()` call contains CJK characters (Chinese/Japanese/Korean), you **MUST** call `canvas.setFont('NotoSerifSC', size)` (or another registered CJK font) before drawing. Using FreeSerif / Helvetica / Times for CJK canvas text = garbled output.
