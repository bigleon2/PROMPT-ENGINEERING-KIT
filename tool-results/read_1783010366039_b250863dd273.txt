   141→   190→   219→   219→   219→
   142→   191→   220→   220→   220→### 3a. Typographic Role Extraction
   143→   192→   221→   221→   221→
   144→   193→   222→   222→   222→Parse raw text and categorize content into visual roles:
   145→   194→   223→   223→   223→
   146→   195→   224→   224→   224→| Role | ReportLab Mapping | Description |
   147→   196→   225→   225→   225→|------|------------------|-------------|
   148→   197→   226→   226→   226→| **Hero / Display** | Cover title (36-42pt) or section opener | 1-5 words, the emotional hook |
   149→   198→   227→   227→   227→| **Kicker / Eyebrow** | Subtitle or section intro (10-12pt, muted) | Tiny context line above main heading |
   150→   199→   228→   228→   228→| **Data Sculpture** | CalloutBox or bold stat block | Extract impactful numbers (97%, $4.2M, -45ms) |
   151→   200→   229→   229→   229→| **Pull Quote** | BlockQuote (italic + left indent 24pt + accent border) | Most provocative sentence, standalone |
   152→   201→   230→   230→   230→| **Body** | Standard paragraph | Everything else |
   153→   202→   231→   231→   231→
   154→   203→   232→   232→   232→**Rule**: Before writing any body paragraph, scan for numbers with units.    156→   205→   234→   234→   234→### 3b. Section Pacing
   157→   206→   235→   235→   235→

   159→   208→   237→   237→   237→- Aim for at least 1 visual element (table/chart/callout) per 2-3 pages of body text

   161→   210→   239→   239→   239→
   162→   211→   240→   240→   240→### 3c. Content Sanitization for External Sources (MANDATORY when source is PDF/OCR/web)
   163→   212→   241→   241→   241→
   164→   213→   242→   242→   242→When content is extracted from an external PDF, OCR output, or web scrape, it may contain invisible garbage characters that cause garbled rendering. **You MUST sanitize the text content before writing it into any `Paragraph()` or `canvas.drawString()`.**
   165→   214→   243→   243→   243→
   166→   215→   244→   244→   244→This is different from `code.sanitize` (which cleans Python source code). Content sanitization cleans the **text data itself**.
   167→   216→   245→   245→   245→
   168→   217→   246→   246→   246→```python
   169→   218→   247→   247→   247→import re
   170→   219→   248→   248→   248→
   171→   220→   249→   249→   249→def content_sanitize(text: str) -> str:
   172→   221→   250→   250→   250→    """Strip invisible/control chars from externally-sourced text."""
   173→   222→   251→   251→   251→    # Remove control chars (except \n \t)
   174→   223→   252→   252→   252→    text = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f\x7f]', '', text)
   175→   224→   253→   253→   253→    # Remove zero-width chars
   176→   225→   254→   254→   254→    text = re.sub(r'[\u200b-\u200f\u2028-\u202f\u2060\ufeff]', '', text)
   177→   226→   255→   255→   255→    # Remove replacement char (upstream encoding error)
   178→   227→   256→   256→   256→    text = text.replace('\ufffd', '')
   179→   228→   257→   257→   257→    # Remove variation selectors
   180→   229→   258→   258→   258→    text = re.sub(r'[\ufe00-\ufe0f]', '', text)
   181→   230→   259→   259→   259→    # Remove private use area chars
   182→   231→   260→   260→   260→    text = re.sub(r'[\ue000-\uf8ff]', '', text)
   183→   232→   261→   261→   261→    return text
   184→   233→   262→   262→   262→```
   185→   234→   263→   263→   263→
   186→   235→   264→   264→   264→**When to use:**
   187→   236→   265→   265→   265→| Source | Must sanitize? |
   188→   237→   266→   266→   266→|--------|---------------|
   189→   238→   267→   267→   267→| User typed text in prompt | No (usually clean) |
   190→   239→   268→   268→   268→| Extracted from PDF via `pdf.py extract.text` | **YES** |
   191→   240→   269→   269→   269→| OCR output | **YES** |
   192→   241→   270→   270→   270→| Web scrape / HTML strip | **YES** |
   193→   242→   271→   271→   271→| LLM-generated content | No (usually clean) |
   194→   243→   272→   272→   272→
   195→   244→   273→   273→   273→⚠️ **Do NOT confuse** `code.sanitize` (Step 6, cleans Python code) with `content_sanitize()` (Step 3, cleans text data). They solve different problems.

   224→   273→   302→   302→   302→## Character Safety Rule (MANDATORY)
   225→   274→   303→   303→   303→
   226→   275→   304→   304→   304→Three rules for safe character handling in ReportLab PDFs:
   227→   276→   305→   305→   305→
   228→   277→   306→   306→   306→**a) Superscripts and subscripts**: Use `<super>`/`<sub>` tags, never raw Unicode superscript/subscript characters (e.g., `\u00b2`, `\u2082`).
   229→   278→   307→   307→   307→
   230→   279→   308→   308→   308→**b) Emoji**: ReportLab cannot render emoji. If content contains emoji, use Creative brief (HTML + Playwright).
   231→   280→   309→   309→   309→
   232→   281→   310→   310→   310→**c) Font fallback for mixed CJK/Latin text (MANDATORY)**: After registering fonts, you **MUST** call `install_font_fallback()` once. This automatically wraps missing-glyph characters in `<font>` tags inside every `Paragraph()`. No manual `<font name="...">` wrapping needed for mixed Chinese-English text. **Skipping this call is the #1 cause of Helvetica garbled text in CJK documents.**
   233→   282→   311→   311→   311→
   234→   283→   312→   312→   312→**d) canvas.drawString / drawRightString CJK font rule (MANDATORY)**: `install_font_fallback()` only works on `Paragraph()` objects. It does NOT affect `canvas.drawString()` / `canvas.drawRightString()` / `canvas.drawCentredString()`. If the text passed to any `canvas.draw*String()` call contains CJK characters (Chinese/Japanese/Korean), you **MUST** call `canvas.setFont('NotoSerifSC', size)` (or another registered CJK font) before drawing. Using FreeSerif / Helvetica / Times for CJK canvas text = garbled output.
   235→   284→   313→   313→   313→
   236→   285→   314→   314→   314→```python
   237→   286→   315→   315→   315→# ❌ WRONG — FreeSerif has no CJK glyphs, '科大讯飞' will be garbled
   238→   287→   316→   316→   316→canvas.setFont('FreeSerif-Italic', 8)
   239→   288→   317→   317→   317→canvas.drawRightString(x, y, 'SZSE: 002230  ·  科大讯飞')
   240→   289→   318→   318→   318→
   241→   290→   319→   319→   319→# ✅ CORRECT — use CJK font for mixed text
   242→   291→   320→   320→   320→canvas.setFont('NotoSerifSC', 8)
   243→   292→   321→   321→   321→canvas.drawRightString(x, y, 'SZSE: 002230  ·  科大讯飞')
   244→   293→   322→   322→   322→
   245→   294→   323→   323→   323→# ✅ ALSO CORRECT — pure English text can use FreeSerif
   246→   295→   324→   324→   324→canvas.setFont('FreeSerif-Italic', 8)
   247→   296→   325→   325→   325→canvas.drawString(x, y, 'Prepared by Strategy Advisory Team')
   248→   297→   326→   326→   326→```

   377→   426→   455→   455→   455→
   378→   427→   456→   456→   456→**For Chinese PDFs:**
   379→   428→   457→   457→   457→- Body text: `NotoSerifSC` (primary) — serif font with sharp, elegant strokes
   380→   429→   458→   458→   458→- Headings: `NotoSerifSC-Bold` (MUST use for Chinese headings) — same family, bolder weight for clear hierarchy

   387→   436→   465→   465→   465→- Body text: `FreeSerif`

   392→   441→   470→   470→   470→**For Mixed Chinese-English PDFs:**
   393→   442→   471→   471→   471→- Call `install_font_fallback()` once after registering fonts - it automatically wraps characters in `<font>` tags so you don't need to do it manually.
   394→   443→   472→   472→   472→- If you still want manual control, you can use `<font name='...'>` tags, but the automatic fallback handles most cases.
   395→   444→   473→   473→   473→

   397→   446→   475→   475→   475→import sys, os
   398→   447→   476→   476→   476→sys.path.insert(0, os.path.join(os.path.dirname(__file__), 'scripts'))

   400→   449→   478→   478→   478→
   401→   450→   479→   479→   479→# 1. Register fonts (as above)
   402→   451→   480→   480→   480→# 2. Install fallback - one line, handles all mixed-language text
   403→   452→   481→   481→   481→install_font_fallback()
   404→   453→   482→   482→   482→
   405→   454→   483→   483→   483→# 3. Just write naturally - no manual <font> wrapping needed!
   406→   455→   484→   484→   484→en_style = ParagraphStyle(name="EN", fontName="FreeSerif", fontSize=10.5, leading=18)

   408→   457→   486→   486→   486→

   414→   463→   492→   492→   492→
   415→   464→   493→   493→   493→**How it works:** `install_font_fallback()` monkey-patches `Paragraph.__init__` to scan each character against the font's `charToGlyph` table. Characters missing from the base font are automatically wrapped in `<font name="FallbackFont">`. The fallback chain is: English fonts → NotoSerifSC (primary) or Noto Sans SC (legacy), Chinese fonts → FreeSerif. For aesthetic optimization, Cyrillic text in NotoSerifSC/Noto Sans SC is automatically routed to FreeSerif (serif looks better for Cyrillic).
